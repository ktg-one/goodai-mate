## 2025-02-27 - Fix SSRF Vulnerability in Website Analyzer
**Vulnerability:** The `/api/analyze-website` endpoint directly accepted a user-provided URL and fetched its content without validating if the URL pointed to an internal/private IP address or local service. This allowed Server-Side Request Forgery (SSRF) which could have been used to port-scan the internal network or access internal services.
**Learning:** Outbound requests initiated by user input (like URL analyzers) must parse the URL using `new URL()` and explicitly reject any private IPv4/IPv6 address blocks, `localhost`, and internal domains. Simple string blocking is insufficient (e.g., `127.0.0.1` can be represented as `2130706433` or `0x7f000001` or `0177.0.0.1` but `new URL()` parses these correctly). Furthermore, attackers can bypass simple hostname checks by using custom domains pointing to internal IPs via DNS A records.
**Prevention:** Always validate user-provided URLs by passing them through a robust parser, and crucially, resolve the hostname using `dns.promises.lookup()` and validate the resulting IP address against private network ranges before fetching.

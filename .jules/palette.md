## 2024-06-24 - Implicit state failure on custom segmented controls
**Learning:** Custom tab and segmented control buttons that rely entirely on visual styling (`className` logic) to indicate their selected state do not communicate this to assistive technologies by default. Standard interactive elements don't inherently adopt a "pressed" or "checked" state unless it's explicitly wired up or they are native inputs (like radio buttons).
**Action:** Always add `aria-pressed={condition}` (for grouped buttons/tabs) or `aria-selected={condition}` / `aria-checked={condition}` (for radiogroups) to these interactive elements to ensure screen readers announce their active state. Also remember to wrap these groups in containers with `role="group"` or `role="radiogroup"` and an `aria-label`.

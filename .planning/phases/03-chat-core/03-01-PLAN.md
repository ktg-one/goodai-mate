**Step 3 — Create `src/app/api/chat/route.ts`:**

Route Handler structure:
1. `export const maxDuration = 60;` — prevent Edge/Node timeout on slow first-token (Pitfall 5)
2. Import `createOpenAICompatible` from `@ai-sdk/openai-compatible`
3. Import `generateText` from `ai`
4. Import `SYSTEM_PROMPT` from `@/lib/chatPersona`
5. Create gateway provider:
   ```typescript
   const gateway = createOpenAICompatible({
     name: 'openai',
     apiKey: process.env.AI_GATEWAY_API_KEY!,
     baseURL: 'https://ai-gateway.vercel.sh/v1',
   });
   ```
6. Export `POST` handler that:
   - Parses request body as JSON, extracts `messages` array
   - Validates: if `!messages` or not array or empty, return 400 with `{ error: 'Messages array required' }`
   - Maps messages: `m.role === 'ai' ? 'assistant' : m.role` (AIBE-06)
   - Calls `generateText` with:
     - `model: gateway('anthropic/claude-sonnet-4-20250514')` (AIBE-03)
     - `system: SYSTEM_PROMPT` (AIBE-04, AIBE-05)
     - `messages: formattedMessages`
     - `maxTokens: 300`
   - Returns `Response.json({ response: result.text })` on success
   - Catches errors:
     - If error has status 429: return `Response.json({ response: "We're getting a lot of traffic right now. Give it a sec and try again." }, { status: 429 })` (AIBE-07)
     - All other errors: return `Response.json({ response: "Something broke on our end. Try again in a moment." }, { status: 500 })` (AIBE-08)
     - Log error to console in all error cases

**What NOT to do:**
- Do NOT use the direct Anthropic SDK (`@anthropic-ai/sdk`) — must go through AI Gateway (AIBE-02)
- Do NOT use `streamText` or streaming for v1 — use `generateText` for full response (streaming is v2 per ECHAT-02)
- Do NOT add CORS headers — same-origin in Next.js Route Handlers, not needed
- Do NOT use `export const runtime = 'edge'` unless testing confirms it works — start with default Node.js runtime to avoid OIDC auth issues locally (Pitfall 4)

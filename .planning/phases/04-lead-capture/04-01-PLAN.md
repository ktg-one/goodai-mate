<task type="auto">
  <name>Task 1: Build LeadCaptureCard component and Web3Forms submission</name>
  <files>src/components/LeadCaptureCard.tsx, src/lib/web3forms.ts</files>
  <action>
    **src/lib/web3forms.ts** — Create the submission utility:
    - Export `submitLead(data: LeadPayload): Promise<boolean>` function
    - LeadPayload type: `{ name: string; business?: string; phone: string; email?: string; problem: string; conversation: string; accessKey: string }`
    - POST to `https://api.web3forms.com/submit` with JSON body containing:
      - `access_key`: from param (WEB3FORMS_KEY, hardcoded in frontend — it's a form key, not a secret)
      - `subject`: `New lead: ${name}`
      - `from_name`: `Good'ai Website`
      - `name`, `business`, `phone`, `email` fields
      - `problem`: first user message from conversation
      - `conversation`: full transcript formatted as `Them: {msg}\nUs: {msg}` alternating
    - Export `serializeTranscript(messages: Message[]): string` — maps messages array to "Them: .../Us: ..." format. Role 'user' = "Them:", role 'ai' = "Us:"
    - On fetch error or non-ok response, return false (do NOT throw — we show success regardless per spec: "On error: still show success")
    - On success, return true

    **src/components/LeadCaptureCard.tsx** — Create the card component:
    - `'use client'` directive
    - Props: `{ messages: Message[]; onCaptured?: () => void }`
    - Use react-hook-form + zod for validation:
      - Schema: name (string, min 1, required), business (string, optional), phone (string, min 1, required, describe as tel), email (string, email format, optional)
    - Use shadcn Card (surface-raised background, accent border), Input, and Button components
    - Layout per spec:
      - Heading: "Want us to look into this?" — 15px, weight 600, text-bright
      - Subtext: "Drop your details and we'll scope it out — no obligation, no runaround." — 13px, text-dim
      - Two rows of fields:
        - Row 1: Name (required, placeholder "Name") + Business (optional, placeholder "Business name")
        - Row 2: Phone (required, type="tel", placeholder "Phone") + Email (optional, placeholder "Email (optional)")
      - Rows use flex-row with gap-2.5, stack to flex-col below 640px (use Tailwind `sm:flex-row flex-col`)
    - Submit button: "Get a callback" — accent background, bg-colored text, 14px, weight 600
      - Disabled state during submission: opacity 50%, text changes to "Sending..."
      - Hover: opacity 90%, translateY(-1px)
    - On submit:
      - Call `submitLead()` with form data + serialized transcript + first user message as problem
      - Regardless of success/failure: transition to success state
      - Set internal `captured` state to true, call `onCaptured` callback
    - Success state: replace card content with checkmark (Unicode 10003) + "Nice one. We'll be in touch within 24 hours." centered, 14px
    - Card entrance animation: fadeUp (translateY 12px to 0) over 0.6s with cubic-bezier(0.23, 1, 0.32, 1) — use Tailwind animate or inline style
    - Subtle gradient overlay: `::before` pseudo or a div with linear-gradient(135deg, rgba(216,106,61,0.06) 0%, transparent 60%) absolutely positioned
    - Style card fields: surface bg, border, 10px radius, 14px font, focus border-color copper at 0.3
    - All inputs and button: `cursor: none` (custom cursor handles visual)
    - WEB3FORMS_KEY: read from `process.env.NEXT_PUBLIC_WEB3FORMS_KEY` (public env var — it's a form key)
  </action>

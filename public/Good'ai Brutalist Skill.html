01/11

Mission

Hold the line

Practical, implementation-ready Good'ai assets with a restrained brutalist layout grammar.

The final output should feel like 1995 direct mail / back-of-the-Yellow-Pages composition, rebuilt with 2026 software: hard-edged, useful, loud in the right places, readable, fast, and unmistakably Good'ai.

Brutalism here does not mean replacing the brand. It means using the existing Good'ai system with harder structure — heavy borders, exposed grid logic, big type against compact utility details, functional blocks over decorative polish.

02/11

When rules collide.

Order of operations

Resolve conflicts in this order — top wins.

  1. Good'ai brand source of truth wins.This document, plus README.md and colors_and_type.css for tokens.
  2. Colour, type, logo, casing, voice are locked.Do not replace them with the generic brutalism palette or fonts.
  3. Accessibility beats aesthetic aggression.Brutalist composition must still pass WCAG 2.2 AA and keyboard usability checks.
  4. Performance beats decoration.No shader backgrounds, full-page pointer loops, or blur-heavy effects.
  5. Clarity beats novelty.If a layout is visually interesting but hard to understand, simplify it.

05/11

The stamp shadow.

Visual signature

Flat offset shadows only. No blurs. This is the brand's signature lift.

  • Ink behind paper is the default.
  • Orange behind ink is for CTA emphasis only.
  • Never use blurred drop shadows.
  • Never stack ink and orange stamp shadows on the same element.
06/11

Composition layer.

Harder structure

Layout rules

  • Grid-first layouts with visible divisions.
  • Rectangular cards, strips, blocks, panels.
  • Let one element be oversized: headline, mark, CTA, score, quote.
  • Keep surroundings calmer so the loud thing has a job.
  • 8-pt rhythm: 4 / 8 / 12 / 16 / 24 / 40 / 64 / 96.
  • Sharp corners by default: 0–8px for cards and buttons.
  • Pills (999px) only for chips, counters, dots, status badges.

Colour-use rules

  • Top-five rule. Colour the first 4–5 elements of a surface; let the rest sit on paper.
  • Cards earn colour. Use yellow, ocean, orange, ink — not just paper.
  • One orange moment per surface, maximum.
  • Ink can be used heavily; paper is the primary canvas.
  • WA Ocean owns a full section, never competes with CTAs.
  • Sticker yellow is for deliberate labels: SALE, FIX THIS, URGENT.
  • No gradients. Ever.
  • No purple, teal, neon blue, tomato red, warm beige.

Default paper-grain background

background:
  repeating-linear-gradient(0deg, rgba(11,11,11,0.025) 0 1px, transparent 1px 4px),
  var(--paper);

Make it useful before you make it clever.

07/11

Components, with states.

Anatomy + variants

Anatomy, variants, states, responsive behaviour, edge cases. Define them or don't ship them.

Component · 01Button

Primary · Secondary

Direct labels, hard edges, stamped shadow.

  • Primary: ink bg, paper text, orange stamp shadow, 2px ink border, 0–8px radius.
  • Hover translate −1, −1. Active translate 2, 2. Shadow flattens.
  • Focus-visible 3px yellow + 2px offset.
  • Disabled paper-3 bg, ink-faint text, no shadow, not-allowed.
  • Loading preserves width. Replace icon with spinner or “Working…”.
  • No pill buttons. No “Submit”.
Component · 02Card

Panels & cards

Eyebrow → headline → body → action.

  • Cards carry colour — paper, yellow, orange, ocean, or ink. Match colour to role.
  • Border 2px ink for primary; 1px ink for quiet cards.
  • Shadow: stamp only.
  • Header: .eyebrow or .mono above .h2/.h3.
  • Long titles wrap. Metadata truncates only when non-essential.
  • Empty state: a direct explanation and one next action.

Workflow · 14

Invoice chase, sorted.

7-day reminders, escalations on day 14, escalation note when overdue.

Component · 03Form

Inputs · labels · errors

Visible labels. No placeholder-only labels. Ever.

  • Inputs: --paper bg, 2px ink border, 4px radius.
  • Focus-visible obvious: yellow outline or orange border + AA contrast.
  • Errors use --warn, text reason, aria-describedby.
  • Helper text is --fg-mute, never low-contrast grey.
  • Touch targets ≥ 44px.
Component · 04Status

Chips, badges, dots

State by text, not by colour alone.

  • .mono or .eyebrow style.
  • Pills (999px) allowed here.
  • Words: Draft, Live, Needs check, Blocked, Done.
  • Colour reinforces, never replaces, the word.

08/11

Accessibility, non-negotiable.

WCAG 2.2 AA

If brutalist tension conflicts with accessibility, reduce the tension.

  • Text contrast passes AA for normal & large text.
  • Focus-visible state on every interactive element.
  • Keyboard users can reach, operate, exit every control.
  • Touch targets ≥ 44px where practical.
  • Form errors announced via text + programmatic association.
  • Motion respects prefers-reduced-motion.
  • No information is conveyed by colour alone.
  • Landmarks are semantic — header, main, section, nav, footer, form.
  • Buttons are buttons; links are links.
  • Images have useful alt text — or empty alt when decorative.

10/11

Anti-patterns.

Do not ship

If you catch yourself doing any of this, stop, and re-read sections 04 and 05.

  • Generic brutalism palette in place of Good'ai colours.
  • Darker Grotesque in place of Fraunces / DM Sans / JetBrains Mono.
  • Gradient backgrounds.
  • Blurred shadows.
  • Glassmorphism.
  • Purple SaaS hero sections.
  • Teal pivots.
  • Emoji as UI icons.
  • Icon fonts.
  • PNG UI icons where SVG / Lucide is right.
  • Stock-photo glass towers.
  • Robot / brain / circuit-board imagery.
  • Cursor-hiding effects.
  • Full-page mouse-tracking RAF loops.
  • Hover-only disclosure.
  • Low-contrast grey-on-paper text.
  • Placeholder-only form labels.
  • Ambiguous CTA labels.
11/11

QA before delivery.

Ship-ready

Every box ticked before the asset leaves your hands.

  • Palette intact — ink, orange, paper, WA Ocean as the core system.
  • Orange appears once per surface, at most.
  • Cards carry brand colour — not just paper.
  • Top-five rule honoured — the rest of the surface sits calm.
  • Fonts are Fraunces, DM Sans, JetBrains Mono.
  • Italic + WONK used for one phrase per surface, max.
  • Brand name casing is Good'ai.
  • g-mark used as supplied — no wordmark, no logo reassembly.
  • Stamp shadows are flat offsets with no blur.
  • Corners are sharp except chips / dots.
  • Layout uses visible grid / block logic.
  • CTAs have default, hover, focus-visible, active, disabled, loading.
  • Forms have visible labels and accessible error text.
  • Motion respects reduced-motion.
  • Text contrast passes WCAG 2.2 AA.
  • Copy avoids the forbidden jargon list.
  • The result is fast, readable, useful.
